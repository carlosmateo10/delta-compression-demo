Welcome to the Delta Compression Demo! This page demonstrates how RFC 9842 shrinks heavy payloads in dynamic websites. See the Activity Graph and Notifications? That simulates heavy, user-specific data that stays constant across your session. When you search, only the articles change. Delta Compression uses your previous page as a dictionary to skip resending the constant data, squeezing the payload down to nearly zero bytes!

Under the Hood

  • The Problem: Dynamic websites resend heavy, user-specific data on every click. Here, your Activity Graph and Notifications simulate this constant, hard-to-compress data overhead.
  • The Mechanism: This server uses RFC 9842 to tell your browser to keep the raw code of your current page in its network cache to use as a reference dictionary.
  • The Result: When you click a link, the server doesn't resend the code for the sidebar. It simply sends the new articles, plus tiny instructions to "copy the rest from the dictionary." This drops the transferred data to near-zero!
